# セキュリティポリシー

## 脆弱性の報告

irori のセキュリティを真剣に受け止めています。脆弱性を発見した場合は、**公開 Issue を作成せず**、非公開で報告してください。

報告手段:

- GitHub の [Security Advisories](https://github.com/syouya0618/irori/security/advisories/new) から非公開で報告する（推奨）

報告には以下を含めていただけると助かります:

- 脆弱性の種類と影響範囲
- 再現手順（該当する場合は PoC）
- 影響を受けるファイル / エンドポイント / コミット

可能な限り速やかに受領確認し、修正方針をご連絡します。

## 対象範囲

このプロジェクトは Supabase（Auth / Database / Storage / Realtime）を利用しています。特に以下に関する報告を歓迎します:

- 行レベルセキュリティ（RLS）ポリシーの不備による他世帯データへのアクセス
- 認証・招待・承認フローの回避
- Server Actions / API 層での認可漏れ
- 保存・転送時の機密情報の漏洩

## 対象外

- 自身で構築した開発環境の設定ミスに起因するもの
- 依存ライブラリの既知 CVE で、本プロジェクトの利用経路では発現しないもの（報告は歓迎しますが優先度は下がります）

## 秘密情報の取り扱い

- `anon` (publishable) key はクライアントに埋め込まれる前提で、RLS により保護されます。露出は脆弱性ではありません。
- `service_role` key・データベースパスワード・アクセストークンは**絶対にコミットしないでください**。万一混入した場合は、該当キーをローテーションした上で報告してください。
